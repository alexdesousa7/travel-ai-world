Run or generate database migrations using Alembic.

Steps to apply existing migrations:
1. Run the upgrade command to bring the database to the latest schema version:
   ```bash
   cd backend
   uv run alembic upgrade head
   ```

Steps to create a new migration after modifying models in `app/models/`:
1. Register your new model by adding it to `app/models/__init__.py` (Alembic's `env.py` imports this package via `import app.models` to discover all tables).
2. Use Alembic to auto-generate a migration script:
   ```bash
   cd backend
   uv run alembic revision --autogenerate -m "Add description of your change"
   ```
3. Review the newly generated script in `backend/alembic/versions/` to verify it looks correct.
4. Apply the migration:
   ```bash
   uv run alembic upgrade head
   ```

> **Tip:** Always review autogenerated migration files before applying — Alembic may miss column renames or subtle type changes.
